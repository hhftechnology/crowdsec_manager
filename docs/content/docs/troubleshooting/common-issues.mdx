# Common Issues and Troubleshooting

This guide covers the most common issues encountered when using CrowdSec Manager with different proxy types and their solutions.

## General Troubleshooting Steps

Before diving into specific issues, follow these general troubleshooting steps:

### 1. Check System Status

```bash
# Check all containers
docker ps -a

# Check container logs
docker logs crowdsec-manager
docker logs crowdsec

# Check system resources
docker stats

# Verify network connectivity
docker network ls
docker network inspect crowdsec-network
```

### 2. Verify Configuration

```bash
# Check environment variables
docker exec crowdsec-manager env | grep PROXY

# Verify configuration files exist
docker exec crowdsec-manager ls -la /data/
docker exec crowdsec ls -la /etc/crowdsec/

# Test API connectivity
curl -f http://localhost:8080/health
```

### 3. Check Logs for Errors

```bash
# Manager logs with timestamps
docker logs -t crowdsec-manager | tail -50

# CrowdSec logs with timestamps
docker logs -t crowdsec | tail -50

# Filter for errors only
docker logs crowdsec-manager 2>&1 | grep -i error
```

## Container Issues

### Container Won't Start

#### Symptoms
- Container exits immediately after starting
- "Container not found" errors
- Port binding failures

#### Common Causes and Solutions

**Port Conflicts**
```bash
# Check what's using the ports
netstat -tulpn | grep :80
netstat -tulpn | grep :443
netstat -tulpn | grep :8080

# Solution: Change ports in docker-compose.yml
ports:
  - "8081:8080"  # Use different host port
```

**Volume Mount Issues**
```bash
# Check volume permissions
ls -la /path/to/volume/

# Fix permissions
sudo chown -R 1000:1000 /path/to/volume/
sudo chmod -R 755 /path/to/volume/
```

**Environment Variable Issues**
```bash
# Check .env file exists and is readable
cat .env

# Verify variables are loaded
docker-compose config
```

**Memory/Resource Constraints**
```bash
# Check available resources
free -h
df -h

# Increase Docker memory limits
# Edit Docker Desktop settings or daemon.json
```

### Container Keeps Restarting

#### Symptoms
- Container status shows "Restarting"
- Repeated startup attempts in logs
- Health check failures

#### Solutions

**Check Health Check Configuration**
```yaml
# Fix health check in docker-compose.yml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
  interval: 30s
  timeout: 10s
  retries: 5
  start_period: 60s  # Give more time for startup
```

**Database Issues**
```bash
# Check database file permissions
docker exec crowdsec-manager ls -la /data/crowdsec-manager.db

# Reset database if corrupted
docker exec crowdsec-manager rm /data/crowdsec-manager.db
docker restart crowdsec-manager
```

**Configuration Errors**
```bash
# Validate configuration
docker exec crowdsec-manager cat /etc/crowdsec-manager/config.yaml

# Check for syntax errors in logs
docker logs crowdsec-manager | grep -i "config\|syntax\|parse"
```

## Proxy-Specific Issues

### Traefik Issues

#### Dynamic Configuration Not Updating

**Symptoms**
- Whitelist changes don't take effect
- Captcha configuration not applied
- Routes not updating

**Solutions**
```bash
# Check dynamic config file permissions
docker exec traefik ls -la /etc/traefik/dynamic_config.yml

# Verify file watcher is working
docker logs traefik | grep "Configuration loaded"

# Force reload
docker restart traefik

# Check for syntax errors
docker exec traefik cat /etc/traefik/dynamic_config.yml | yaml-lint
```

#### Bouncer Plugin Not Loading

**Symptoms**
- Plugin not found errors
- Bouncer middleware not working
- 500 errors on protected routes

**Solutions**
```bash
# Check plugin installation
docker exec traefik ls -la /plugins-local/

# Verify plugin configuration
docker exec traefik cat /etc/traefik/traefik_config.yml | grep -A 10 plugins

# Check plugin logs
docker logs traefik | grep -i bouncer

# Reinstall plugin
docker exec traefik rm -rf /plugins-local/
docker restart traefik
```

#### SSL Certificate Issues

**Symptoms**
- SSL certificate errors
- ACME challenge failures
- Certificate not auto-renewing

**Solutions**
```bash
# Check ACME configuration
docker exec traefik cat /etc/traefik/acme.json

# Verify domain DNS
nslookup yourdomain.com

# Check certificate resolver
docker logs traefik | grep -i acme

# Force certificate renewal
docker exec traefik rm /etc/traefik/acme.json
docker restart traefik
```

### Nginx Proxy Manager Issues

#### NPM Admin Interface Not Accessible

**Symptoms**
- Cannot access NPM on port 81
- Login page not loading
- Database connection errors

**Solutions**
```bash
# Check NPM container status
docker logs nginx

# Verify database connectivity
docker exec nginx mysql -h npm-db -u npm -p

# Reset admin credentials
docker exec nginx npx npm-cli reset-password admin@example.com newpassword

# Check port binding
docker port nginx
```

#### Bouncer Integration Not Working

**Symptoms**
- cs-nginx-bouncer not blocking IPs
- LAPI connection failures
- Configuration not loading

**Solutions**
```bash
# Check bouncer installation
docker exec nginx which cs-nginx-bouncer

# Verify bouncer configuration
docker exec nginx cat /etc/crowdsec/bouncers/crowdsec-nginx-bouncer.conf

# Test LAPI connectivity
docker exec nginx curl -H "X-Api-Key: your-key" http://crowdsec:8080/v1/decisions

# Restart bouncer service
docker exec nginx systemctl restart crowdsec-nginx-bouncer
```

#### Log Parsing Issues

**Symptoms**
- Logs not appearing in CrowdSec Manager
- Parsing errors in CrowdSec logs
- Log files not accessible

**Solutions**
```bash
# Check log file permissions
docker exec nginx ls -la /data/logs/

# Verify log format
docker exec nginx tail -f /data/logs/proxy-host-1.log

# Check CrowdSec acquisition
docker exec crowdsec cat /etc/crowdsec/acquis.yaml

# Test log parsing
docker exec crowdsec cscli parsers test -f /var/log/nginx/access.log
```

### Caddy Issues

#### Bouncer Module Not Available

**Symptoms**
- "module not found" errors
- Bouncer middleware not recognized
- Build failures

**Solutions**
```bash
# Check if bouncer module is built-in
docker exec caddy caddy list-modules | grep crowdsec

# Rebuild Caddy with bouncer plugin
docker build -t caddy-crowdsec -f Dockerfile.caddy .

# Verify Caddyfile syntax
docker exec caddy caddy validate --config /etc/caddy/Caddyfile
```

#### Automatic HTTPS Issues

**Symptoms**
- Certificate provisioning failures
- HTTP not redirecting to HTTPS
- Certificate validation errors

**Solutions**
```bash
# Check certificate status
docker exec caddy caddy trust

# Verify domain configuration
docker exec caddy caddy config --config /etc/caddy/Caddyfile

# Check ACME logs
docker logs caddy | grep -i acme

# Force certificate renewal
docker exec caddy caddy reload --config /etc/caddy/Caddyfile
```

### HAProxy Issues

#### SPOA Bouncer Connection Failures

**Symptoms**
- SPOA connection timeouts
- Bouncer not responding
- 503 errors on protected routes

**Solutions**
```bash
# Check SPOA bouncer status
docker logs crowdsec-haproxy-bouncer

# Test SPOA connectivity
docker exec haproxy nc -zv crowdsec-haproxy-bouncer 9999

# Verify SPOE configuration
docker exec haproxy cat /usr/local/etc/haproxy/spoe-crowdsec.conf

# Check HAProxy SPOA status
docker exec haproxy echo "show servers state" | socat stdio /var/run/haproxy.sock
```

#### Configuration Validation Errors

**Symptoms**
- HAProxy won't start
- Configuration syntax errors
- SPOE parsing failures

**Solutions**
```bash
# Validate HAProxy configuration
docker exec haproxy haproxy -c -f /usr/local/etc/haproxy/haproxy.cfg

# Check SPOE configuration
docker exec haproxy haproxy -c -f /usr/local/etc/haproxy/spoe-crowdsec.conf

# Verify file permissions
docker exec haproxy ls -la /usr/local/etc/haproxy/
```

### Database Issues

#### Database Corruption

**Symptoms**
- SQLite database errors
- Data inconsistencies
- Performance degradation

**Solutions**

**Check Database Integrity**
```bash
# Check database file
docker exec crowdsec ls -la /var/lib/crowdsec/data/crowdsec.db

# Test database connectivity
docker exec crowdsec sqlite3 /var/lib/crowdsec/data/crowdsec.db ".tables"

# Check for corruption
docker exec crowdsec sqlite3 /var/lib/crowdsec/data/crowdsec.db "PRAGMA integrity_check;"
```

**Repair or Reset Database**
```bash
# Backup current database
docker cp crowdsec:/var/lib/crowdsec/data/crowdsec.db ./crowdsec-backup.db

# Reset database (WARNING: loses all data)
docker exec crowdsec rm /var/lib/crowdsec/data/crowdsec.db
docker restart crowdsec

# Restore from backup if needed
docker cp ./crowdsec-backup.db crowdsec:/var/lib/crowdsec/data/crowdsec.db
```

## Network and Connectivity Issues

### DNS Resolution Problems

#### Symptoms
- Container name resolution failures
- External API connection issues
- Hub update failures

#### Solutions

**Check DNS Configuration**
```bash
# Test DNS resolution
docker exec crowdsec nslookup google.com

# Check container DNS
docker exec crowdsec cat /etc/resolv.conf

# Test container-to-container resolution
docker exec crowdsec-manager nslookup crowdsec
```

**Fix DNS Issues**
```yaml
# Add DNS configuration to docker-compose.yml
services:
  crowdsec:
    dns:
      - 8.8.8.8
      - 8.8.4.4
```

### Firewall and Port Issues

#### Symptoms
- External access blocked
- Container communication failures
- Port binding errors

#### Solutions

**Check Firewall Rules**
```bash
# Check iptables rules
sudo iptables -L

# Check UFW status (Ubuntu)
sudo ufw status

# Allow required ports
sudo ufw allow 80
sudo ufw allow 443
sudo ufw allow 8080
```

**Docker Network Issues**
```bash
# Recreate Docker network
docker network rm crowdsec-network
docker network create crowdsec-network

# Check Docker daemon
sudo systemctl status docker

# Restart Docker if needed
sudo systemctl restart docker
```

## Performance Issues

### High Resource Usage

#### Symptoms
- High CPU or memory usage
- Slow response times
- Container OOM kills

#### Solutions

**Monitor Resource Usage**
```bash
# Check container resources
docker stats

# Check system resources
htop
free -h
df -h
```

**Optimize Configuration**
```yaml
# Add resource limits to docker-compose.yml
services:
  crowdsec:
    deploy:
      resources:
        limits:
          memory: 512M
          cpus: '0.5'
```

**Database Optimization**
```bash
# Optimize CrowdSec database
docker exec crowdsec sqlite3 /var/lib/crowdsec/data/crowdsec.db "VACUUM;"

# Clean old decisions
docker exec crowdsec cscli decisions delete --all --older-than 7d
```

### Slow Log Processing

#### Symptoms
- Log parsing delays
- High parser CPU usage
- Backlog of unprocessed logs

#### Solutions

**Optimize Log Processing**
```yaml
# Increase parser routines in CrowdSec config
crowdsec_service:
  parser_routines: 4  # Increase from default 1
```

**Log Rotation**
```bash
# Configure log rotation
# Add to /etc/logrotate.d/application
/var/log/app/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 644 app app
}
```

## Migration Issues

### Upgrade Problems

#### Symptoms
- Configuration compatibility issues
- Database migration failures
- Feature regressions

#### Solutions

**Backup Before Upgrade**
```bash
# Backup all data
docker cp crowdsec:/var/lib/crowdsec/data ./crowdsec-backup
docker cp crowdsec-manager:/data ./manager-backup

# Backup configurations
docker cp crowdsec:/etc/crowdsec ./config-backup
```

**Handle Migration Issues**
```bash
# Check migration logs
docker logs crowdsec-manager | grep -i migration

# Reset to clean state if needed
docker-compose down -v
docker-compose up -d

# Restore from backup
# (restore backed up data and configurations)
```

### Proxy Type Changes

#### Symptoms
- Features not working after proxy change
- Configuration conflicts
- Service startup failures

#### Solutions

**Clean Migration Process**
```bash
# Stop all services
docker-compose down

# Clean volumes if needed
docker volume prune

# Update configuration
# Edit .env file with new PROXY_TYPE

# Deploy with new proxy type
docker-compose --profile new-proxy up -d
```

When seeking help, collect this information:

```bash

# Container status
```

### Log Analysis

**Enable Debug Logging**
```bash
# Enable debug mode in manager
LOG_LEVEL=debug

# Enable debug in CrowdSec
# Edit /etc/crowdsec/config.yaml
log_level: debug
```

**Structured Log Analysis**
```bash
# Filter logs by component
docker logs crowdsec-manager | grep "proxy"
docker logs crowdsec | grep "LAPI"

# Check for specific errors
docker logs crowdsec-manager | grep -i "error\|fail\|exception"

# Monitor real-time logs
docker logs -f crowdsec-manager
```
