# Frequently Asked Questions (FAQ)

This FAQ covers the most common questions about CrowdSec Manager's multi-proxy architecture and deployment options.

## General Questions

### What is CrowdSec Manager's multi-proxy architecture?

CrowdSec Manager's multi-proxy architecture allows you to use CrowdSec security features with different reverse proxy solutions. Instead of being limited to Traefik, you can now choose from:

- **Traefik** - Full feature support with advanced integrations
- **Nginx Proxy Manager** - Log parsing and bouncer integration  
- **Caddy** - Basic bouncer integration with automatic HTTPS
- **HAProxy** - SPOA bouncer integration for high performance
- **Zoraxy** - Experimental support for newer proxy technologies
- **Standalone** - CrowdSec only, without reverse proxy integration

### Can I switch between proxy types after deployment?

**No, proxy type switching after initial deployment is not supported.** This is by design to maintain architectural simplicity and deployment consistency. The proxy type is set during initial deployment via the `PROXY_TYPE` environment variable and stored permanently.

If you need to change proxy types, you must:
1. Backup your current configuration and data
2. Deploy a new instance with the desired proxy type
3. Migrate your security policies and configurations manually

### Is my existing Traefik setup compatible?

**Yes, existing Traefik setups are fully compatible.** The system automatically detects existing Traefik configurations and migrates them seamlessly:

- All current settings are preserved
- API endpoints maintain backward compatibility
- Existing integrations continue working
- All Traefik features remain available

No manual intervention is required for existing Traefik users.

## Deployment Questions

### What's the difference between single and separate compose modes?

**Single Mode (`COMPOSE_MODE=single`)**:
- All services in one `docker-compose.yml` file
- Uses Docker Compose profiles to manage different proxy types
- Easier to manage and deploy
- Recommended for most users

```bash
# Deploy Traefik mode
docker-compose --profile traefik up -d

# Deploy Nginx mode  
docker-compose --profile nginx up -d
```

**Separate Mode (`COMPOSE_MODE=separate`)**:
- CrowdSec core in `crowdsec-compose.yml`
- Proxy-specific services in separate files (`proxy-traefik.yml`, etc.)
- More modular and flexible
- Better for complex deployments

```bash
# Deploy with separate files
docker-compose -f crowdsec-compose.yml -f proxy-traefik.yml up -d
```

### How do I choose the right proxy type for my needs?

**Choose Traefik if:**
- You need full whitelist management at the proxy level
- You want Cloudflare Turnstile captcha protection
- You require advanced log parsing and analysis
- You want to use Pangolin and Gerbil add-ons
- You're migrating from an existing Traefik setup

**Choose Nginx Proxy Manager if:**
- You prefer NPM's web-based configuration interface
- You need basic log monitoring capabilities
- You want CrowdSec protection with minimal setup
- You're already using NPM in your infrastructure

**Choose Caddy if:**
- You prefer Caddy's automatic HTTPS capabilities
- You want a lightweight proxy with CrowdSec protection
- You need minimal configuration overhead
- You're building a simple web service setup

**Choose HAProxy if:**
- You require HAProxy's advanced load balancing features
- You need high-performance proxy capabilities
- You're working with enterprise-grade requirements
- You have existing HAProxy expertise

**Choose Standalone if:**
- You don't use a reverse proxy
- You want CrowdSec protection for direct services
- You're building API-only applications
- You need maximum simplicity

### Can I use multiple proxy types in the same deployment?

**No, each CrowdSec Manager instance supports only one proxy type.** However, you can deploy multiple CrowdSec Manager instances with different proxy types if needed:

```yaml
# Multiple instances example
services:
  # Traefik instance
  crowdsec-manager-traefik:
    image: crowdsec-manager:latest
    environment:
      - PROXY_TYPE=traefik
    ports:
      - "8080:8080"

  # Nginx instance  
  crowdsec-manager-nginx:
    image: crowdsec-manager:latest
    environment:
      - PROXY_TYPE=nginx
    ports:
      - "8081:8080"
```

## Configuration Questions

### How do I configure environment variables?

Create a `.env` file in your deployment directory:

```bash
# Core Configuration
PROXY_TYPE=traefik                    # Required: proxy type
PROXY_ENABLED=true                    # Enable proxy integration
COMPOSE_MODE=single                   # Deployment mode

# Proxy-Specific Settings
PROXY_CONTAINER_NAME=traefik          # Proxy container name
TRAEFIK_DYNAMIC_CONFIG=/etc/traefik/dynamic_config.yml
TRAEFIK_STATIC_CONFIG=/etc/traefik/traefik_config.yml

# Domain Configuration
DOMAIN=yourdomain.com                 # Your domain
EMAIL=admin@yourdomain.com            # Email for SSL certificates

# CrowdSec Configuration
CROWDSEC_BOUNCER_KEY=your-key-here    # Bouncer API key
CROWDSEC_LAPI_URL=http://crowdsec:8080

# Database and Storage
DATABASE_PATH=/data/crowdsec-manager.db
BACKUP_PATH=/backups
```

### How do I manage SSL certificates?

**Traefik (Automatic)**:
```yaml
# Traefik handles SSL automatically via Let's Encrypt
certificatesResolvers:
  letsencrypt:
    acme:
      email: your-email@domain.com
      storage: /etc/traefik/acme.json
      httpChallenge:
        entryPoint: web
```

**Nginx Proxy Manager (Web Interface)**:
- Use NPM's web interface to request SSL certificates
- Supports Let's Encrypt and custom certificates
- Automatic renewal handled by NPM

**Caddy (Automatic)**:
```caddyfile
# Caddy handles SSL automatically
yourdomain.com {
    reverse_proxy backend:8080
    # SSL is automatic - no configuration needed
}
```

**HAProxy (Manual)**:
```bash
# Combine certificate and key for HAProxy
cat yourdomain.com.crt yourdomain.com.key > yourdomain.com.pem
```

### How do I configure custom domains?

**Update Environment Variables:**
```bash
DOMAIN=yourdomain.com
EMAIL=admin@yourdomain.com
```

**Proxy-Specific Configuration:**

*Traefik:*
```yaml
# In dynamic_config.yml
http:
  routers:
    crowdsec-manager:
      rule: "Host(`manager.yourdomain.com`)"
      service: crowdsec-manager
      tls:
        certResolver: letsencrypt
```

*Nginx Proxy Manager:*
- Use NPM web interface to create proxy hosts
- Set domain name and SSL certificate
- Point to `crowdsec-manager:8080`

*Caddy:*
```caddyfile
manager.yourdomain.com {
    reverse_proxy crowdsec-manager:8080
}
```

## Feature-Specific Questions

### How does whitelist management work?

**Traefik (Full Support)**:
- Dynamic whitelist management through CrowdSec Manager UI
- Automatically updates Traefik dynamic configuration
- Supports IP addresses and CIDR ranges
- Changes take effect immediately

**Other Proxies (Limited/No Support)**:
- Use proxy-specific whitelist features (NPM Access Lists, HAProxy ACLs)
- CrowdSec whitelist still works for CrowdSec decisions
- No automatic proxy-level whitelist management

### What captcha providers are supported?

**Traefik Only**:
- **Cloudflare Turnstile** - Recommended, privacy-focused
- **Configuration**: Site key and secret key required
- **Modes**: Managed, Non-interactive, Invisible
- **Themes**: Light and Dark
- **Integration**: Automatic middleware creation

**Other Proxies**:
- No built-in captcha support
- Use proxy-specific plugins or external services
- Custom implementation required

### How does log parsing work?

**Traefik (Advanced)**:
- Comprehensive access log parsing
- Real-time log analysis and statistics
- Attack pattern detection
- Geographic distribution analysis

**Nginx Proxy Manager (Basic)**:
- NPM access log parsing from `/data/logs/proxy-host-*.log`
- Basic request statistics
- Limited analysis capabilities

**Other Proxies**:
- No proxy log parsing
- CrowdSec system logs only
- Custom log integration possible

### What are Pangolin and Gerbil add-ons?

**Pangolin** (Traefik-specific):
- Enhanced security features for Traefik
- Advanced threat detection
- Custom security rules
- Integration with CrowdSec decisions

**Gerbil** (Traefik-specific):
- Advanced log analysis capabilities
- Real-time log streaming
- Custom alert rules
- Integration with notification systems

**Availability**: Only available with Traefik proxy type

```bash
# Enable add-ons with Traefik
docker-compose --profile traefik --profile pangolin --profile gerbil up -d
```

## Security Questions

### How secure is the multi-proxy architecture?

**Security Features**:
- **Isolation**: Each proxy adapter is isolated and sandboxed
- **Validation**: All configuration inputs are validated
- **Authentication**: Secure API key management for bouncers
- **Encryption**: HTTPS/TLS for all communications
- **Auditing**: Comprehensive logging and monitoring

**Best Practices**:
- Use strong, unique API keys
- Enable HTTPS for all services
- Regular security updates
- Monitor logs for suspicious activity
- Implement proper backup strategies

### How are API keys managed?

**CrowdSec Bouncer Keys**:
```bash
# Generate new bouncer key
docker exec crowdsec cscli bouncers add manager-bouncer

# List existing bouncers
docker exec crowdsec cscli bouncers list

# Delete bouncer
docker exec crowdsec cscli bouncers delete manager-bouncer
```

**Key Rotation**:
1. Generate new API key
2. Update environment variables
3. Restart affected services
4. Delete old API key

**Security Recommendations**:
- Use unique keys for each bouncer
- Rotate keys regularly (quarterly recommended)
- Store keys securely (environment variables, secrets management)
- Monitor key usage and access

### What data is collected and stored?

**CrowdSec Manager Stores**:
- Configuration settings and preferences
- Security decisions and alerts
- System health and performance metrics
- User activity logs (authentication, configuration changes)

**CrowdSec Core Stores**:
- Security decisions and their metadata
- Attack patterns and signatures
- Machine learning data for threat detection
- Log parsing results and statistics

**Data Retention**:
- Security decisions: 7 days default (configurable)
- System logs: 30 days default (configurable)
- Configuration backups: User-defined retention
- Performance metrics: 90 days default

**Privacy Considerations**:
- No personal data collection
- IP addresses are hashed for privacy
- Compliance with GDPR and privacy regulations
- Data can be anonymized or purged on request

## Performance Questions

### What's the performance impact of different proxy types?

**Performance Characteristics**:

**Traefik**:
- **Overhead**: Low to moderate
- **Features**: High (full feature set impacts performance)
- **Scalability**: Good for medium to large deployments
- **Resource Usage**: Moderate CPU and memory

**Nginx Proxy Manager**:
- **Overhead**: Low
- **Features**: Basic (minimal performance impact)
- **Scalability**: Excellent for high-traffic sites
- **Resource Usage**: Low CPU and memory

**Caddy**:
- **Overhead**: Very low
- **Features**: Minimal (very low performance impact)
- **Scalability**: Good for small to medium deployments
- **Resource Usage**: Very low CPU and memory

**HAProxy**:
- **Overhead**: Very low
- **Features**: Basic (SPOA adds minimal overhead)
- **Scalability**: Excellent for enterprise deployments
- **Resource Usage**: Low CPU and memory

**Standalone**:
- **Overhead**: Minimal (no proxy overhead)
- **Features**: CrowdSec only
- **Scalability**: Depends on application
- **Resource Usage**: Minimal

### How do I optimize performance?

**General Optimization**:
```yaml
# Resource limits in docker-compose.yml
services:
  crowdsec:
    deploy:
      resources:
        limits:
          memory: 512M
          cpus: '0.5'
```

**CrowdSec Optimization**:
```yaml
# Increase parser routines for high log volume
crowdsec_service:
  parser_routines: 4

# Optimize database settings
db_config:
  flush:
    max_items: 10000
    max_age: 3d
```

**Proxy-Specific Optimization**:
- **Traefik**: Optimize middleware order, use efficient matchers
- **Nginx**: Configure worker processes, enable caching
- **Caddy**: Enable HTTP/2 and HTTP/3, use compression
- **HAProxy**: Tune connection limits, optimize SPOA timeouts

## Troubleshooting Questions

### Why isn't my proxy container starting?

**Common Causes**:
1. **Port Conflicts**: Ports 80, 443, or 8080 already in use
2. **Configuration Errors**: Invalid syntax in configuration files
3. **Volume Permissions**: Incorrect file permissions for mounted volumes
4. **Resource Constraints**: Insufficient memory or CPU
5. **Network Issues**: Docker network configuration problems

**Debugging Steps**:
```bash
# Check container logs
docker logs your-proxy-container

# Verify port availability
netstat -tulpn | grep :80

# Check configuration syntax
docker exec your-proxy-container config-validation-command

# Test with minimal configuration
# Remove custom configurations and test with defaults
```

### Why isn't CrowdSec blocking malicious IPs?

**Common Causes**:
1. **Bouncer Not Connected**: Bouncer not properly connected to LAPI
2. **Configuration Issues**: Incorrect bouncer configuration
3. **Scenario Problems**: Scenarios not triggering or not installed
4. **Network Issues**: Communication problems between components

**Debugging Steps**:
```bash
# Check bouncer status
docker exec crowdsec cscli bouncers list

# Test LAPI connectivity
curl -H "X-Api-Key: your-key" http://crowdsec:8080/v1/decisions

# Check active decisions
docker exec crowdsec cscli decisions list

# Verify scenario installation
docker exec crowdsec cscli scenarios list
```

### How do I reset to a clean state?

**Complete Reset** (WARNING: Loses all data):
```bash
# Stop all services
docker-compose down

# Remove volumes (loses all data)
docker-compose down -v

# Remove containers and images
docker system prune -a

# Redeploy
docker-compose up -d
```

**Partial Reset** (Preserves some data):
```bash
# Reset only CrowdSec Manager database
docker exec crowdsec-manager rm /data/crowdsec-manager.db
docker restart crowdsec-manager

# Reset only CrowdSec decisions
docker exec crowdsec cscli decisions delete --all
```

## Migration and Upgrade Questions

### How do I upgrade to a newer version?

**Standard Upgrade Process**:
```bash
# Backup current data
docker cp crowdsec:/var/lib/crowdsec/data ./crowdsec-backup
docker cp crowdsec-manager:/data ./manager-backup

# Pull latest images
docker-compose pull

# Restart services
docker-compose up -d

# Verify upgrade
curl http://localhost:8080/health
```

**Major Version Upgrades**:
1. Review changelog for breaking changes
2. Test upgrade in non-production environment
3. Backup all data and configurations
4. Follow version-specific upgrade instructions
5. Verify all features work correctly

### How do I migrate from other security solutions?

**From Fail2ban**:
1. Document existing Fail2ban rules and configurations
2. Map Fail2ban filters to CrowdSec scenarios
3. Configure log acquisition for same log sources
4. Test CrowdSec detection before disabling Fail2ban
5. Gradually migrate rules and monitor effectiveness

**From ModSecurity**:
1. Analyze existing ModSecurity rules
2. Implement equivalent CrowdSec scenarios
3. Configure application log parsing
4. Test detection accuracy and performance
5. Migrate rules incrementally

**From Commercial WAF**:
1. Document current security policies
2. Implement equivalent CrowdSec scenarios
3. Configure comprehensive logging
4. Test thoroughly in parallel deployment
5. Migrate traffic gradually with monitoring
