# Standalone Mode Setup Guide

This guide covers setting up CrowdSec Manager in standalone mode, providing CrowdSec security features without reverse proxy integration.

## Overview

Standalone mode provides CrowdSec core functionality without proxy integration:

- ❌ **Whitelist Management** - CrowdSec whitelist only (no proxy integration)
- ❌ **Captcha Protection** - Not available (no proxy middleware)
- ❌ **Log Parsing** - CrowdSec logs only (no proxy logs)
- ❌ **Bouncer Integration** - No proxy bouncer (CrowdSec core only)
- ✅ **Health Monitoring** - CrowdSec components and API monitoring
- ❌ **Add-ons Support** - Traefik-specific features not available

## Use Cases

Standalone mode is ideal for:

- **API-only Applications**: Services that don't need a reverse proxy
- **Direct Service Exposure**: Applications with built-in HTTP servers
- **Microservices**: Individual services with their own security needs
- **Development/Testing**: Simplified setup for development environments
- **CrowdSec Evaluation**: Testing CrowdSec features without proxy complexity

## Prerequisites

- Docker and Docker Compose installed
- Basic understanding of CrowdSec concepts
- Network access for CrowdSec Hub updates
- Understanding of your application's security requirements

## Quick Start

### 1. Environment Configuration

Create your `.env` file:

```bash
# Proxy Configuration
PROXY_TYPE=standalone
PROXY_ENABLED=false
COMPOSE_MODE=single

# CrowdSec Configuration
CROWDSEC_LAPI_URL=http://crowdsec:8080
CROWDSEC_MACHINE_ID=crowdsec-manager-standalone
CROWDSEC_PASSWORD=your-secure-password

# Manager Configuration
MANAGER_HOST=0.0.0.0
MANAGER_PORT=8080
MANAGER_LOG_LEVEL=info

# Database Configuration
DATABASE_PATH=/data/crowdsec-manager.db
BACKUP_PATH=/backups
```

### 2. Deploy with Docker Compose

```bash
# Start standalone mode (no proxy profile needed)
docker-compose up -d

# Or explicitly specify standalone
docker-compose --profile standalone up -d
```

### 3. Verify Installation

1. Access CrowdSec Manager: `http://your-server:8080`
2. Check Dashboard → CrowdSec Health
3. Verify CrowdSec container is running
4. Confirm LAPI connectivity

  profiles_path: /etc/crowdsec/profiles.yaml

## Application Integration

### Direct Application Protection

For applications without reverse proxy:

#### API Protection
```python
# Python Flask example with CrowdSec integration
from flask import Flask, request, abort
import requests

app = Flask(__name__)

def check_crowdsec_decision(ip):
    """Check if IP is banned by CrowdSec"""
    try:
        response = requests.get(
            f"http://crowdsec:8080/v1/decisions?ip={ip}",
            headers={"X-Api-Key": "your-bouncer-key"},
            timeout=1
        )
        return len(response.json()) > 0
    except:
        return False  # Fail open

@app.before_request
def check_security():
    client_ip = request.environ.get('HTTP_X_FORWARDED_FOR', request.remote_addr)
    if check_crowdsec_decision(client_ip):
        abort(403)

@app.route('/api/data')
def get_data():
    return {"data": "protected content"}
```

### Custom Bouncer Implementation

Create a simple bouncer for your application:

```go
// Simple Go bouncer example
package main

import (
    "encoding/json"
    "fmt"
    "net/http"
    "time"
)

type Decision struct {
    IP       string    `json:"ip"`
    Duration string    `json:"duration"`
    Until    time.Time `json:"until"`
    Reason   string    `json:"reason"`
}

type Bouncer struct {
    apiKey  string
    lapiURL string
    cache   map[string]time.Time
}

func (b *Bouncer) isBlocked(ip string) bool {
    // Check cache first
    if until, exists := b.cache[ip]; exists {
        if time.Now().Before(until) {
            return true
        }
        delete(b.cache, ip)
    }

    // Query CrowdSec LAPI
    resp, err := http.Get(fmt.Sprintf("%s/v1/decisions?ip=%s", b.lapiURL, ip))
    if err != nil {
        return false // Fail open
    }
    defer resp.Body.Close()

    var decisions []Decision
    if err := json.NewDecoder(resp.Body).Decode(&decisions); err != nil {
        return false
    }

    if len(decisions) > 0 {
        b.cache[ip] = decisions[0].Until
        return true
    }

    return false
}

func (b *Bouncer) middleware(next http.Handler) http.Handler {
    return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
        clientIP := r.Header.Get("X-Forwarded-For")
        if clientIP == "" {
            clientIP = r.RemoteAddr
        }

        if b.isBlocked(clientIP) {
            http.Error(w, "Access Denied", http.StatusForbidden)
            return
        }

        next.ServeHTTP(w, r)
    })
}
```

### Custom Monitoring

Implement custom monitoring for your setup:

```python
# Python monitoring script
import requests
import time
import logging

class CrowdSecMonitor:
    def __init__(self, lapi_url, api_key):
        self.lapi_url = lapi_url
        self.api_key = api_key
        
    def check_lapi_health(self):
        """Check LAPI health"""
        try:
            response = requests.get(
                f"{self.lapi_url}/health",
                timeout=5
            )
            return response.status_code == 200
        except:
            return False
    
    def get_decision_count(self):
        """Get current decision count"""
        try:
            response = requests.get(
                f"{self.lapi_url}/v1/decisions",
                headers={"X-Api-Key": self.api_key},
                timeout=5
            )
            return len(response.json())
        except:
            return -1
    
    def monitor_loop(self):
        """Main monitoring loop"""
        while True:
            lapi_healthy = self.check_lapi_health()
            decision_count = self.get_decision_count()
            
            logging.info(f"LAPI Health: {lapi_healthy}, Decisions: {decision_count}")
            
            if not lapi_healthy:
                logging.error("LAPI is not responding!")
                # Send alert
            
            time.sleep(60)

# Usage
monitor = CrowdSecMonitor("http://crowdsec:8080", "your-api-key")
monitor.monitor_loop()
```

## Limitations and Considerations

### Feature Limitations
- **No Proxy Integration**: Cannot manage proxy-level whitelists or captcha
- **Limited Log Sources**: Only system and application logs (no proxy logs)
- **No Automatic Blocking**: Requires custom bouncer implementation
- **Manual Integration**: Applications must implement CrowdSec checking

### Security Considerations
- **Fail-Open Strategy**: Applications should fail open if CrowdSec is unavailable
- **API Key Security**: Protect bouncer API keys and rotate regularly
- **Network Security**: Secure communication between applications and CrowdSec
- **Monitoring**: Implement comprehensive monitoring for security gaps

### Performance Considerations
- **LAPI Response Time**: Applications must handle LAPI timeouts gracefully
- **Caching Strategy**: Implement decision caching to reduce LAPI load
- **Resource Usage**: Monitor CrowdSec resource usage with high log volumes
- **Database Performance**: Regular database maintenance for optimal performance

1. **Assess Current Setup**

4. **Verify Functionality**
   - Test CrowdSec decision enforcement

### From External Security to CrowdSec

1. **Parallel Deployment**
   - Deploy CrowdSec alongside existing security

   - Gradually migrate traffic

2. **Feature Mapping**
   - Map existing security rules to CrowdSec scenarios
   - Implement custom scenarios for specific needs
   - Configure appropriate decision durations

3. **Gradual Migration**
   - Start with monitoring mode
   - Enable blocking for low-risk scenarios
   - Gradually expand coverage
