# Nginx Proxy Manager Setup Guide

This guide covers setting up CrowdSec Manager with Nginx Proxy Manager (NPM), including log parsing and bouncer integration.

## Overview

Nginx Proxy Manager mode provides essential CrowdSec integration:

- ❌ **Whitelist Management** - Not supported (managed via NPM interface)
- ❌ **Captcha Protection** - Not supported (use NPM's built-in features)
- ✅ **Log Parsing** - NPM access log monitoring and analysis
- ✅ **Bouncer Integration** - cs-nginx-bouncer integration
- ✅ **Health Monitoring** - Container and bouncer status checking
- ❌ **Add-ons Support** - Traefik-specific features not available

## Prerequisites

- Docker and Docker Compose installed
- Basic understanding of Nginx Proxy Manager
- Domain name for SSL certificates (recommended)
- Existing NPM setup or willingness to configure it

## Quick Start

### 1. Environment Configuration

Create your `.env` file:

```bash
# Proxy Configuration
PROXY_TYPE=nginx
PROXY_ENABLED=true
PROXY_CONTAINER_NAME=nginx
COMPOSE_MODE=single

# NPM-specific settings
NPM_DATA_PATH=/data
NPM_LOG_PATH=/data/logs
NPM_CONFIG_PATH=/data/nginx

# Database configuration (NPM)
NPM_DB_MYSQL_HOST=npm-db
NPM_DB_MYSQL_PORT=3306
NPM_DB_MYSQL_USER=npm
NPM_DB_MYSQL_PASSWORD=npm_password
NPM_DB_MYSQL_NAME=npm

# CrowdSec Configuration
CROWDSEC_BOUNCER_KEY=your-bouncer-key-here
```

### 2. Deploy with Docker Compose

```bash
# Start nginx profile
docker-compose --profile nginx up -d

# Or with separate files
docker-compose -f crowdsec-compose.yml -f proxy-nginx.yml up -d
```

### 3. Initial NPM Setup

1. Access NPM interface: `http://your-server:81`
2. Default login: `admin@example.com` / `changeme`
3. Change default credentials immediately
4. Configure your first proxy host

### 4. Verify Installation

1. Access CrowdSec Manager: `https://manager.yourdomain.com`
2. Check Dashboard → Proxy Health
3. Verify NPM container is running
4. Confirm bouncer connection status

## Detailed Configuration

### Docker Compose Configuration

Your NPM service configuration:

```yaml
services:
  nginx:
    image: 'jc21/nginx-proxy-manager:latest'
    container_name: nginx
    profiles: ["nginx"]
    restart: unless-stopped
    ports:
      - '80:80'
      - '443:443'
      - '81:81'  # NPM Admin interface
    environment:
      - DB_MYSQL_HOST=npm-db
      - DB_MYSQL_PORT=3306
      - DB_MYSQL_USER=npm
      - DB_MYSQL_PASSWORD=npm_password
      - DB_MYSQL_NAME=npm
    volumes:
      - npm_data:/data
      - npm_letsencrypt:/etc/letsencrypt
      - /var/log/nginx:/var/log/nginx:rw
    depends_on:
      - npm-db
    networks:
      - crowdsec-network

  npm-db:
    image: 'mysql:8.0'
    container_name: npm-db
    profiles: ["nginx"]
    restart: unless-stopped
    environment:
      - MYSQL_ROOT_PASSWORD=root_password
      - MYSQL_DATABASE=npm
      - MYSQL_USER=npm
      - MYSQL_PASSWORD=npm_password
    volumes:
      - npm_db:/var/lib/mysql
    networks:
      - crowdsec-network
```

### CrowdSec Bouncer Integration

#### Installing cs-nginx-bouncer

The bouncer should be installed in the NPM container:

```dockerfile
# Custom NPM image with CrowdSec bouncer
FROM jc21/nginx-proxy-manager:latest

# Install cs-nginx-bouncer
RUN apt-get update && \
    apt-get install -y wget && \
    wget https://github.com/crowdsecurity/cs-nginx-bouncer/releases/latest/download/cs-nginx-bouncer-linux-amd64.tgz && \
    tar -xzf cs-nginx-bouncer-linux-amd64.tgz && \
    ./install.sh

# Copy bouncer configuration
COPY crowdsec-nginx-bouncer.conf /etc/crowdsec/bouncers/
```

#### Bouncer Configuration

Create `/etc/crowdsec/bouncers/crowdsec-nginx-bouncer.conf`:

```ini
# CrowdSec Nginx Bouncer Configuration
[crowdsec_config]
# CrowdSec LAPI URL
lapi_url = http://crowdsec:8080
# Bouncer API key
lapi_key = your-bouncer-key-here
# Update frequency
update_frequency = 10s

[nginx_config]
# Nginx configuration path
nginx_config = /data/nginx/custom/crowdsec.conf
# Reload command
reload_cmd = nginx -s reload

[logging]
# Log level: trace, debug, info, warn, error, fatal, panic
log_level = info
# Log file path
log_file = /var/log/crowdsec-nginx-bouncer.log
```

#### Nginx Configuration Integration

The bouncer creates `/data/nginx/custom/crowdsec.conf`:

```nginx
# CrowdSec bouncer configuration
# This file is automatically generated - do not edit manually

# Blocked IPs map
map $remote_addr $crowdsec_blocked {
    default 0;
    # Blocked IPs will be added here automatically
    include /etc/crowdsec/bouncers/blocked_ips.map;
}

# Block requests from banned IPs
server {
    # This configuration is included in all proxy hosts
    if ($crowdsec_blocked) {
        return 403 "Access Denied by CrowdSec";
    }
}
```

## Feature Configuration

### Log Parsing and Analysis

NPM access logs are automatically parsed by CrowdSec Manager:

#### Log Location
NPM stores access logs at:
- `/data/logs/proxy-host-*.log` - Individual proxy host logs
- `/data/logs/default.log` - Default server logs
- `/data/logs/manager.log` - NPM manager logs

#### Log Format Configuration

Configure NPM to use a CrowdSec-compatible log format:

```nginx
# Custom log format for CrowdSec
log_format crowdsec_format '$remote_addr - $remote_user [$time_local] '
                          '"$request" $status $body_bytes_sent '
                          '"$http_referer" "$http_user_agent" '
                          '$request_time $upstream_response_time';

# Apply to proxy hosts
access_log /data/logs/proxy-host-$proxy_host_id.log crowdsec_format;
```

#### CrowdSec Acquisition Configuration

Update `/etc/crowdsec/acquis.yaml`:

```yaml
# NPM access logs
filenames:
  - /data/logs/proxy-host-*.log
labels:
  type: nginx
  service: nginx-proxy-manager

# NPM error logs
filenames:
  - /data/logs/error.log
labels:
  type: nginx-error
  service: nginx-proxy-manager
```

## NPM Integration Best Practices

### Proxy Host Configuration

When creating proxy hosts in NPM:

#### 1. SSL Configuration
- Always use SSL certificates (Let's Encrypt recommended)
- Enable HTTP to HTTPS redirection
- Use strong SSL/TLS settings

#### 2. Security Headers
Add security headers in NPM's "Advanced" tab:

```nginx
# Security headers
add_header X-Frame-Options "SAMEORIGIN" always;
add_header X-Content-Type-Options "nosniff" always;
add_header X-XSS-Protection "1; mode=block" always;
add_header Referrer-Policy "strict-origin-when-cross-origin" always;
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

# Include CrowdSec protection
include /data/nginx/custom/crowdsec.conf;
```

#### 3. Rate Limiting
Configure rate limiting per proxy host:

```nginx
# Rate limiting
limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
limit_req zone=api burst=20 nodelay;
```

### Database Optimization

Optimize NPM's MySQL database:

```sql
-- Optimize NPM database performance
SET GLOBAL innodb_buffer_pool_size = 256M;
SET GLOBAL max_connections = 200;
SET GLOBAL query_cache_size = 64M;

-- Regular maintenance
OPTIMIZE TABLE proxy_hosts;
OPTIMIZE TABLE certificates;
OPTIMIZE TABLE access_lists;
```

## Limitations and Workarounds

### Whitelist Management
**Limitation**: No direct proxy-level whitelist management
**Workaround**: Use NPM's Access Lists feature:

1. Go to NPM Admin → Access Lists
2. Create new access list with allowed IPs
3. Apply to specific proxy hosts
4. Use CrowdSec whitelist for global protection

### Captcha Protection
**Limitation**: No built-in captcha middleware
**Workaround**: Use NPM's custom locations:

```nginx
# Custom captcha location
location /captcha-challenge {
    # Implement custom captcha logic
    # Or integrate with external captcha service
}
```

### Advanced Security Features
**Limitation**: Limited compared to Traefik mode
**Workaround**: Combine NPM features with CrowdSec:

- Use NPM's rate limiting + CrowdSec decisions
- Combine NPM access lists + CrowdSec whitelist
- Use NPM's custom headers + CrowdSec protection

## Migration from Other Proxies

### From Traefik to NPM

1. **Export Traefik Configuration**
   ```bash
   # Backup Traefik routes and services
   docker exec traefik cat /etc/traefik/dynamic_config.yml > traefik-backup.yml
   ```

2. **Create NPM Proxy Hosts**
   - Manually recreate each Traefik route as NPM proxy host
   - Configure SSL certificates
   - Set up security headers

3. **Update CrowdSec Manager**
   ```bash
   # Change proxy type
   PROXY_TYPE=nginx
   docker-compose up -d
   ```

4. **Verify Functionality**
   - Test all proxy hosts work correctly
   - Verify CrowdSec integration
   - Check log parsing functionality

### From Apache/Other to NPM

1. **Document Current Configuration**
   - List all virtual hosts/sites
   - Note SSL certificate locations
   - Document any custom configurations

2. **Set Up NPM**
   - Create proxy hosts for each site
   - Configure SSL certificates
   - Test functionality

3. **Integrate CrowdSec**
   - Deploy CrowdSec Manager with NPM
   - Configure bouncer integration
   - Set up log monitoring
